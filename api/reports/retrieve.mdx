## Overview

The Retrieve Report endpoint fetches complete details for a single report, including full client and user information. This endpoint is useful for displaying report details, downloading PDFs, and accessing complete analytics data.

## Endpoint

```
GET /api/reports/{id}
```

## Authentication

Requires authentication via NextAuth.js session cookie. Users can only retrieve reports they own.

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier (CUID) of the report to retrieve.
</ParamField>

## Response

### Success Response (200 OK)

Returns a detailed report object with nested client and user information:

<ResponseField name="report" type="object">
  Complete report object with all fields
  
  <Expandable title="Report Properties">
    <ResponseField name="id" type="string">
      Unique report identifier (CUID format)
    </ResponseField>
    
    <ResponseField name="title" type="string">
      Report title formatted as: `{ReportType} Report - {ClientName} ({StartDate} to {EndDate})`
    </ResponseField>
    
    <ResponseField name="status" type="enum">
      Current report processing status from the `ReportStatus` enum:
      - `PENDING` - Queued for processing
      - `PROCESSING` - Currently being generated
      - `COMPLETED` - Successfully generated and available
      - `FAILED` - Generation failed (see errorMessage)
    </ResponseField>
    
    <ResponseField name="data" type="object">
      Comprehensive report data object stored as JSON in the database:
      
      <Expandable title="Data Object Structure">
        **Basic Information:**
        - `clientName` (string) - Client display name
        - `startDate` (string) - ISO 8601 reporting period start
        - `endDate` (string) - ISO 8601 reporting period end
        - `reportType` (enum) - "executive" | "standard" | "custom"
        - `generatedAt` (string) - ISO 8601 generation timestamp
        
        **Analytics Data:**
        - `gscData` (object) - Google Search Console metrics:
          - `clicks`, `impressions`, `ctr`, `position` - Aggregate metrics
          - `topQueries` - Array of top performing search queries
          - `dailyData` - Time-series data for trend charts
        - `ga4Data` (object) - Google Analytics 4 metrics (flexible schema):
          - Required: `users`, `sessions`, `bounceRate`, `conversions`
          - Optional: 20+ additional metrics like `newUsers`, `engagementRate`, etc.
          - `topLandingPages` - Array of top pages with metrics
          - `deviceBreakdown` - Mobile/desktop/tablet usage
        - `pageSpeedData` (object | null) - Performance metrics:
          - `mobile`, `desktop` - Scores and Core Web Vitals
          - `opportunities` - Optimization suggestions
        
        **Custom Configuration:**
        - `selectedMetrics` (array) - Metric IDs for custom reports
        - `customFields` (array) - Custom content sections
        
        **Branding:**
        - `agencyName` (string) - Agency display name
        - `agencyLogo` (string) - Logo URL
        
        **AI Insights:**
        - `insights` (array) - AI-generated insight objects from Claude
        
        **Metadata:**
        - `dataSourceInfo` (object) - Information about included data:
          - `hasGSC`, `hasGA4`, `hasPageSpeed` - Boolean flags
          - `metricsCount` - Number of GA4 metrics included
          - `aiInsightsAvailable` - Whether AI insights were generated
      </Expandable>
    </ResponseField>
    
    <ResponseField name="pdfUrl" type="string">
      Public URL to the generated PDF file in Vercel Blob storage. Use this URL to download or display the PDF.
    </ResponseField>
    
    <ResponseField name="pdfSize" type="number">
      Size of the PDF file in bytes. Typical range: 150,000 - 2,000,000 bytes (150KB - 2MB).
    </ResponseField>
    
    <ResponseField name="processingStartedAt" type="string">
      ISO 8601 timestamp when report generation started.
    </ResponseField>
    
    <ResponseField name="processingCompletedAt" type="string">
      ISO 8601 timestamp when report generation completed successfully.
    </ResponseField>
    
    <ResponseField name="generationTimeMs" type="number">
      Total processing time in milliseconds. Typical range: 15,000 - 45,000ms (15-45 seconds).
    </ResponseField>
    
    <ResponseField name="errorMessage" type="string">
      Error description if status is FAILED, otherwise null.
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp when the report record was created in the database.
    </ResponseField>
    
    <ResponseField name="updatedAt" type="string">
      ISO 8601 timestamp of the last update to the report record.
    </ResponseField>
    
    <ResponseField name="clientId" type="string">
      Foreign key reference to the associated client (CUID).
    </ResponseField>
    
    <ResponseField name="userId" type="string">
      Foreign key reference to the user who generated the report (CUID).
    </ResponseField>
    
    <ResponseField name="aiInsights" type="array">
      Array of AI-generated insight objects from Claude API:
      ```typescript
      {
        id: string
        title: string
        description: string
        priority: 'high' | 'medium' | 'low'
        category: 'keyword' | 'technical' | 'content' | 'performance'
        recommendations: string[]
      }
      ```
    </ResponseField>
    
    <ResponseField name="aiInsightsSource" type="enum">
      Source of the insights:
      - `ai` - Generated by Claude API
      - `rule-based` - Generated by fallback rule-based system
      - `fallback` - Default insights (AI service unavailable)
    </ResponseField>
    
    <ResponseField name="aiInsightsGeneratedAt" type="string">
      ISO 8601 timestamp when AI insights were generated.
    </ResponseField>
    
    <ResponseField name="aiTokensUsed" type="number">
      Token count consumed during AI generation. Used for cost tracking and billing.
    </ResponseField>
    
    <ResponseField name="aiCostUsd" type="number">
      Estimated cost in USD for AI generation based on Claude API pricing.
    </ResponseField>
    
    <ResponseField name="aiError" type="string">
      Error message if AI insights generation failed, otherwise null.
    </ResponseField>
    
    <ResponseField name="client" type="object">
      Associated client information with full details:
      ```typescript
      {
        id: string
        name: string
        domain: string
        contactEmail: string | null
        contactName: string | null
      }
      ```
    </ResponseField>
    
    <ResponseField name="user" type="object">
      Report owner (agency) information with branding details:
      ```typescript
      {
        id: string
        name: string | null
        email: string
        companyName: string | null
        primaryColor: string
        logo: string | null
      }
      ```
    </ResponseField>
  </Expandable>
</ResponseField>

## Use Cases

### Display Report Details
Fetch complete report information for display on a report detail page, including all analytics data and AI insights.

### Download PDF
Use the `pdfUrl` field to provide a download link or embed the PDF in an iframe.

### Regenerate Report
Extract the `data` field to regenerate a report with the same configuration and parameters.

### Performance Analysis
Analyze `generationTimeMs`, `pdfSize`, and AI cost metrics for optimization and billing.

### Troubleshooting
Check `status`, `errorMessage`, and `aiError` fields to diagnose failed report generations.

### Client Reporting
Combine report data with client and user information for comprehensive client dashboards.

## Security

This endpoint implements ownership verification:
1. Requires valid authentication session
2. Verifies that the report belongs to the authenticated user
3. Returns 404 if report doesn't exist OR doesn't belong to the user (preventing enumeration)

## Data Persistence

All report data is stored in PostgreSQL with:
- **JSONB storage** for flexible `data` field
- **Indexed queries** on `userId`, `clientId`, and `status`
- **Cascade deletion** when user or client is deleted
- **Automatic timestamps** for `createdAt` and `updatedAt`

## Delete Report

### Endpoint

```
DELETE /api/reports/{id}
```

### Description

Permanently deletes a report and updates the associated client's report count.

### Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the report to delete.
</ParamField>

### Response

<ResponseExample>
```json 200 OK
{
  "success": true
}
```

```json 401 Unauthorized
{
  "error": "Unauthorized"
}
```

```json 404 Not Found
{
  "error": "Report not found or unauthorized"
}
```

```json 500 Internal Server Error
{
  "error": "Failed to delete report"
}
```
</ResponseExample>

### Example Request

```bash
curl -X DELETE https://your-domain.com/api/reports/clxreport123456 \
  -H "Cookie: next-auth.session-token=..."
```

### Behavior

1. Verifies report exists and belongs to authenticated user
2. Deletes the report record from database
3. Decrements the client's `totalReportsGenerated` count
4. Does NOT delete the PDF file from Vercel Blob (manual cleanup required)

**Note:** Consider implementing blob storage cleanup in a background job for production use.
